import/export

import React,{ Component } from 'react'

在 ES6 当中引入了一组个新的关键字 import/export，import用于引入其他库文件（注意，以前是用require引入），所有import的文件都是从其他文件导出
(export),如果使用export default语句导出，这样的方法每个文件只能导出一个。

// myFunction.js
export default function() {
    console.log('This is default function!');
}
// index.js
import myFunction from 'myFunction';

仅用 export 导出的方法，在使用时，则需要把它们包含在大括号里：

// myAnotherFunction.js
export const foo = 'bar';
export function bar() {
    console.log('foo');
}
// index.js
import { foo, bar } from 'myAnotherFunction';

const

const关键字在es6语法中被用来声明常量，const声明的实际是一个指针，指针的位置不能改变，但是指针的值是可以操作的。

// 定义一个空数组Arr
const Arr = [];
// 输出空数组的内容
console.log(Arr);
// []
// 通过push可以向数组添加内容
Arr.push(1);
// 输出新的Arr
console.log(Arr);
// [1]
// 但要将 Arr 指向一个新的数组就会报错
Arr = [];
// Uncaught TypeError: Assignment to constant variable.

let

let关键字在es6语法中用来声明变量。它的用法类似于var，但是所声明的变量，只在let命令所在的代码块内有效。
for循环的计数器，就很合适使用let命令。

for (let i = 0; i < 10; i++) {
  // ...
}

console.log(i);
// ReferenceError: i is not defined

上面代码中，计数器i只在for循环体内有效，在循环体外引用就会报错。

下面的代码如果使用var，最后输出的是10。

for (let i = 0; i < 10; i++) {
  // ...
}

console.log(i);
// ReferenceError: i is not defined

另一个有意思的例子，
var i指向的就是全局的i，也就是说，所有数组a的成员里面的i，指向的都是同一个i，导致运行时输出的是最后一轮的i的值，也就是 10。
变量i是let声明的，当前的i只在本轮循环有效，所以每一次循环的i其实都是一个新的变量，所以最后输出的是6。

var a = [];
for (var i = 0; i < 10; i++) {
  a[i] = function () {
    console.log(i);
  };
}
a[6](); // 10

var a = [];
for (let i = 0; i < 10; i++) {
  a[i] = function () {
    console.log(i);
  };
}
a[6](); // 6

class

在 ES6 当中，我们可以通过 class 关键字继承 React组件 的原型来创建组件。

class APP extends Component {
  constructor(props) {
    super(props)
  }
}

这里的super其实是为了调用其继承父类的构造函数，由此将子类的this初始化，这样我们才能在后面的代码中调动this.state,this.props

箭头函数（注意函数体内的this对象，就是定义时所在的对象，而不是使用时所在的对象。）

ES6 允许使用“箭头”（=>）定义函数。

var f = v => v;

// 等同于
var f = function (v) {
  return v;
};

如果箭头函数的代码块部分多于一条语句，就要使用大括号将它们括起来，并且使用return语句返回。

var sum = (num1, num2) => { return num1 + num2; }

由于大括号被解释为代码块，所以如果箭头函数直接返回一个对象，必须在对象外面加上括号，否则会报错。

// 报错
let getTempItem = id => { id: id, name: "Temp" };

// 不报错
let getTempItem = id => ({ id: id, name: "Temp" });

箭头函数使得表达更加简洁。箭头函数的作用是简化回调函数

const isEven = n => n % 2 === 0;
const square = n => n * n;

总结

1、在 ES6 中新加入的 import 和 export 可以方便我们很好地组织引用代码的各个模块。（所以代码可以更好地模块化和组件化）
2、const和let的用法和区别，const定义常量，实际是一个指针，如果是数组或者对象，可以通过push等方法改变值，let定义变量，都是块级作用域，不存在变量提升
3、class类的实现主要是继承父级的原型，super主要是用来调用父级的构造函数，以使得我们能够在子类中正确的获取this
4、箭头函数，相当于是缩写函数的语法糖，同时，在react中使用箭头函数能实现制动绑定this的功能
